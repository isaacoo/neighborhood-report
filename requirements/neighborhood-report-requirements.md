# 이사 갈 동네 리포트 서비스 요구사항 정의서

## 1. 프로젝트 개요

### 1.1 서비스 비전

정부 제공 공공 API를 활용해 사용자가 이사 후보 동네를 객관적으로 비교하고, 주거비, 생활 편의, 교통, 환경, 안전 인프라 관점에서 의사결정을 돕는 동네 분석 리포트 서비스

### 1.2 핵심 가치 제안

**이사 예정자 관점**

- 여러 후보 지역의 주거비와 생활 인프라를 한 화면에서 비교
- 실거래가, 대기질, 병원, 학교, 교통 등 흩어진 정보를 리포트로 통합
- 사용자의 생활 우선순위에 맞춘 동네 장단점 확인

**1인 가구 및 가족 관점**

- 병원, 약국, 대중교통, 학교, 공원 등 생활 필수 시설 접근성 확인
- 야간 귀가, 출퇴근, 육아, 건강 민감도 등 생활 패턴별 체크포인트 제공
- 단순 평점이 아닌 근거 데이터 기반 설명 제공

**시장 차별화**

- 민간 후기나 광고성 매물 정보가 아닌 공공 데이터 기반 분석
- 후보지별 점수 산정 근거를 투명하게 표시
- 부동산 투자 추천이 아닌 실제 거주 적합성 중심의 리포트 제공

## 2. 서비스 구성

### 2.1 주요 구성 요소

- **사용자용 인터페이스**: 이사 후보지를 검색하고 동네 리포트를 확인하는 웹 UI
- **후보지 비교 화면**: 여러 지역의 주요 지표와 점수를 표/차트로 비교하는 화면
- **리포트 생성 시스템**: 공공 API 데이터를 수집, 정규화, 점수화하여 리포트 생성
- **공공 API 연동 모듈**: 국토교통부, 기상청, 한국환경공단, 행정안전부 등 외부 API 호출 관리
- **데이터 저장소**: 지역, 후보지, API 응답 캐시, 리포트 결과 저장
- **운영자용 인터페이스**: API 상태, 캐시 갱신, 점수 가중치 설정을 관리하는 화면

### 2.2 공공 데이터 활용 후보

- **주소 및 지역 기준 데이터**: 도로명주소, 법정동/행정동 코드, 좌표 변환 정보
- **주거비 데이터**: 국토교통부 실거래가 공개 데이터(아파트 매매, 전월세 등)
- **환경 데이터**: 한국환경공단 에어코리아 대기오염 정보
- **기상 데이터**: 기상청 단기예보 및 생활기상 관련 데이터
- **의료 인프라 데이터**: 병원, 약국, 응급의료기관 정보
- **교육 인프라 데이터**: 학교 기본 정보 및 위치 정보
- **교통 및 생활시설 데이터**: 지하철역, 버스정류장, 공공시설, 공원 등 지역별 가용 데이터

### 2.3 데이터 처리 원칙

- 공공 API 응답은 원본값과 가공값을 분리하여 저장
- 모든 점수는 산정 기준과 사용된 지표를 함께 표시
- API 호출 실패 시 이전 캐시 데이터와 데이터 기준 시점을 명확히 표시
- 데이터가 부족한 지역은 낮은 점수를 부여하지 않고 "데이터 부족" 상태로 표시

## 3. 핵심 기능 요구사항

### 3.1 사용자용 기능 (User Features)

#### 3.1.1 지역 검색 및 후보지 등록

**목적**: 사용자가 이사 후보 동네를 쉽게 찾고 비교 대상으로 저장할 수 있도록 지원

**기능 요구사항**:

- 주소, 동 이름, 구/시군구명으로 지역 검색
- 검색 결과에서 법정동 또는 행정동 단위 후보지 선택
- 후보지 별칭 입력
  - 예: 회사 근처, 부모님 집 근처, 예산 후보 1
- 후보지는 최대 5개까지 등록
- 동일 지역 중복 등록 방지
- 지역 검색 실패 시 대체 검색어 안내

**데이터 관리**:

- 후보지명, 행정구역 코드, 좌표, 등록 시각 저장
- 비로그인 MVP에서는 브라우저 로컬 저장소에 임시 저장
- 후보지 삭제 시 관련 리포트 캐시도 사용자 화면에서 제거

#### 3.1.2 주거비 및 실거래가 분석

**목적**: 후보 지역의 최근 주거비 수준과 거래 흐름을 객관적으로 파악

**기능 요구사항**:

- 최근 실거래가 조회
  - 매매 실거래가
  - 전월세 실거래가
- 면적대별 가격 요약
  - 최저가
  - 중앙값
  - 최고가
  - 거래 건수
- 최근 3개월, 6개월, 12개월 기준 조회 기간 선택
- 거래 건수가 부족한 경우 신뢰도 낮음 표시
- 거래 상세 정보 표시
  - 거래 월
  - 건물명 또는 단지명
  - 전용면적
  - 거래 금액
  - 층
  - 건축연도

**UI/UX 요구사항**:

- 가격 단위는 만원/억원 단위로 읽기 쉽게 표시
- 거래 건수와 기준 기간을 가격 요약 옆에 항상 표시
- 가격 추세는 단순 선형 차트 또는 월별 막대 차트로 표시

#### 3.1.3 생활 인프라 분석

**목적**: 후보 지역 주변의 생활 편의성과 필수 시설 접근성을 평가

**기능 요구사항**:

- 후보지 중심 좌표 기준 반경별 시설 수 조회
  - 500m
  - 1km
  - 2km
- 주요 시설 카테고리 표시
  - 병원
  - 약국
  - 학교
  - 공원
  - 공공기관
  - 지하철역 또는 주요 대중교통 거점
- 시설별 거리, 이름, 주소 표시
- 카테고리별 접근성 점수 산정
- 사용자가 중요 카테고리를 선택하면 종합 점수 가중치에 반영

**데이터 관리**:

- 정적 시설 데이터는 캐시 저장 후 주기적으로 갱신
- 좌표가 없는 시설은 거리 계산 대상에서 제외하고 데이터 누락으로 기록

#### 3.1.4 환경 및 기상 리스크 분석

**목적**: 대기질, 날씨, 생활 환경 정보를 통해 건강 및 외부 활동 리스크를 확인

**기능 요구사항**:

- 가장 가까운 대기측정소 기준 대기질 정보 표시
  - 미세먼지(PM10)
  - 초미세먼지(PM2.5)
  - 오존
  - 통합대기환경지수
- 기상청 단기예보 기반 최근 날씨 및 단기 예보 표시
- 폭염, 강수, 한파 등 생활 불편 가능성 요약
- 데이터 기준 시각과 측정소명을 명확히 표시
- 환경 민감 사용자용 설명 제공
  - 비염/천식 민감
  - 아이 동반
  - 야외활동 빈도 높음

**UI/UX 요구사항**:

- 좋음/보통/나쁨/매우나쁨 상태를 색상과 텍스트로 함께 표시
- 단일 지표가 전체 지역 품질을 대표하지 않는다는 안내 문구 표시

#### 3.1.5 안전 및 생활 리스크 체크

**목적**: 재난 대응, 응급 접근성, 야간 생활 편의 등 공개 데이터로 확인 가능한 안전 요소를 점검

**기능 요구사항**:

- 응급의료기관, 소방서, 경찰서 등 안전 인프라 접근성 표시
- 민방위 대피소 또는 재난 대피시설 정보가 제공되는 경우 위치 표시
- 침수, 산사태 등 재난 관련 공개 데이터가 제공되는 지역은 리스크 참고 정보 표시
- 범죄율처럼 민감하거나 지역 낙인을 만들 수 있는 지표는 MVP 점수 산정에서 제외
- 안전 정보는 절대적 판단이 아닌 참고 지표로 표시

**데이터 관리**:

- 데이터 출처와 기준일을 안전 지표마다 표시
- 공식 API로 확인되지 않은 위험 정보는 사용하지 않음

#### 3.1.6 동네 종합 점수 및 리포트 생성

**목적**: 후보 지역의 장단점을 사용자가 이해하기 쉬운 형태의 리포트로 제공

**기능 요구사항**:

- 후보지별 종합 점수 산정
- 주요 점수 항목
  - 주거비 적정성
  - 생활 인프라
  - 교통 접근성
  - 환경 쾌적성
  - 안전 인프라
  - 데이터 신뢰도
- 점수별 산정 근거 표시
- 장점, 주의점, 데이터 부족 항목 요약
- 사용자 생활 유형별 리포트 문장 생성
  - 출퇴근 중심
  - 육아 중심
  - 1인 가구 중심
  - 건강 민감형
- 리포트 생성 실패 시 원인과 재시도 버튼 표시

**리포트 정보 포함 항목**:

- 후보지 기본 정보
- 데이터 기준 시각
- 사용된 공공 API 목록
- 지표별 원본 요약값
- 점수 산정 결과
- 주의사항 및 면책 안내

#### 3.1.7 후보지 비교

**목적**: 여러 후보 동네를 한 화면에서 비교하고 최종 후보를 좁힐 수 있도록 지원

**기능 요구사항**:

- 후보지별 종합 점수 비교
- 지표별 비교 테이블 표시
- 후보지 간 강점/약점 자동 요약
- 사용자가 선택한 중요 조건 기준으로 재정렬
- 후보지 상세 리포트로 이동
- 후보지 비교 결과 로컬 저장

**UI/UX 요구사항**:

- 모바일에서도 2개 후보지는 가로 스크롤 없이 비교 가능
- 3개 이상 후보지는 표 가로 스크롤 또는 카드 전환 방식 사용
- 가장 우수한 지표와 데이터 부족 지표를 시각적으로 구분

### 3.2 운영자용 기능 (Admin Features)

#### 3.2.1 공공 API 연동 상태 관리

**목적**: 외부 공공 API의 연결 상태와 오류를 관리

**기능 요구사항**:

- API 키 및 엔드포인트 설정
- API별 상태 확인
  - 정상
  - 지연
  - 실패
  - 할당량 초과
- 최근 오류 메시지 및 발생 시각 표시
- 수동 재시도 기능
- API별 응답 시간 기록

**보안 요구사항**:

- API 키는 서버 환경변수 또는 비밀 저장소에 보관
- 클라이언트 화면에 API 키 노출 금지
- 운영자 화면은 관리자 인증 후 접근

#### 3.2.2 데이터 캐시 및 갱신 관리

**목적**: 공공 API 호출 비용과 응답 지연을 줄이고 안정적인 리포트 생성을 지원

**기능 요구사항**:

- API 응답 캐시 저장
- 데이터 유형별 캐시 만료 시간 설정
  - 실거래가: 24시간
  - 시설 정보: 7일
  - 대기질/날씨: 1시간
  - 행정구역 코드: 30일
- 특정 지역 캐시 수동 갱신
- 오래된 캐시 데이터 표시
- 캐시 갱신 실패 시 기존 캐시 유지

**데이터 관리**:

- 원본 응답 저장 시 API명, 요청 파라미터, 기준 시각 함께 저장
- 캐시 삭제 시 관련 리포트 재생성 필요 상태로 변경

#### 3.2.3 점수 가중치 관리

**목적**: 서비스 정책에 따라 동네 종합 점수 산정 방식을 조정

**기능 요구사항**:

- 지표별 기본 가중치 설정
- 사용자 생활 유형별 가중치 프리셋 관리
- 점수 계산식 버전 관리
- 가중치 변경 시 기존 리포트와 신규 리포트 구분 표시
- 잘못된 가중치 입력 방지

**데이터 검증**:

- 전체 가중치 합계 검증
- 필수 지표 누락 방지
- 음수 가중치 입력 금지

## 4. MVP 개발 범위

### 핵심 기능 (필수)

**목표**: 공공 API 기반 후보 동네 비교 리포트의 핵심 가치 검증

**사용자용**:

- 지역 검색 및 후보지 등록
- 후보지 최대 3개 비교
- 실거래가 요약 조회
- 대기질 및 단기예보 요약 조회
- 병원, 약국, 학교, 교통 거점 등 생활 인프라 반경 조회
- 종합 점수 및 지표별 점수 표시
- 후보지별 장점/주의점 요약 리포트

**운영자용**:

- 공공 API 키 설정
- API 연동 상태 확인
- 데이터 캐시 저장 및 만료 처리
- 기본 점수 가중치 관리

### MVP 제외 가능 기능 (후속)

- 사용자 계정 기반 저장
- PDF 다운로드
- 지도 기반 상세 탐색
- 부동산 매물 연동
- 실시간 교통 소요시간 계산
- 개인 맞춤형 장기 추천 알림

## 부록

### A. 용어 정의

- **MVP**: Minimum Viable Product (최소 기능 제품)
- **API**: Application Programming Interface (애플리케이션 프로그래밍 인터페이스)
- **공공 API**: 정부, 공공기관, 지자체가 제공하는 공개 데이터 연동 인터페이스
- **후보지**: 사용자가 이사를 고려하는 동네 또는 주소 기반 지역
- **법정동**: 법률상 주소 체계에 사용되는 동 단위 행정구역
- **행정동**: 주민센터 등 행정 운영을 위해 사용하는 동 단위 행정구역
- **리포트 기준 시각**: 리포트 생성에 사용된 데이터가 수집 또는 갱신된 시각
- **데이터 신뢰도**: 거래 건수, 데이터 최신성, API 응답 완성도를 반영한 참고 지표
