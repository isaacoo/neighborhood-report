# Business Rules — U-4 user-features

## BR-1. 후보지 등록 제한
- 최대 5개까지 등록 가능 (`MAX_CANDIDATES = 5`)
- 동일 법정동 코드 중복 등록 불가
- 별칭 최대 30자 (`MAX_ALIAS_LENGTH = 30`)
- 좌표는 한국 영토 내 (lat 33~39.5, lon 124~132)

## BR-2. 검색어 제한
- 최소 1자, 최대 100자 (`MAX_SEARCH_QUERY_LENGTH = 100`)
- 빈 문자열 거부
- 결과 0건 시 대체 검색어 안내 제공

## BR-3. 실거래가 조회 기간
- 3개월 / 6개월 / 12개월 선택 가능
- 기본값: 6개월
- 1개월 lag 적용 (공공 데이터 반영 지연 고려)

## BR-4. 신뢰도 낮음 판정
- 특정 면적대 거래 건수 < 5건 → `isLowReliability = true`
- UI에서 경고 아이콘 + "신뢰도 낮음" 라벨 표시

## BR-5. 데이터 부족 처리
- 외부 API 실패 + 캐시 없음 → 해당 카테고리 "데이터 부족" 표시
- 임의의 낮은 점수 부여 금지
- 가용 데이터로만 종합 점수 정규화

## BR-6. 대기질 등급 판정
- KHAI ≤ 50: good (좋음)
- KHAI ≤ 100: moderate (보통)
- KHAI ≤ 250: bad (나쁨)
- KHAI > 250: veryBad (매우나쁨)
- 단조성 보장 (지수 ↑ → 등급 ↑)

## BR-7. 측정소 매칭
- 시도 단위 측정소 목록 조회
- 시군구명과 측정소명 매칭 (예: "강남구" → 측정소명에 "강남" 포함)
- 매칭 실패 시 첫 번째 측정소 fallback

## BR-8. 시군구 좌표 매핑
- 행안부 법정동코드 API는 좌표 미제공
- 서울 25개 자치구 대표 좌표 정적 매핑 테이블 사용
- 매핑 미등록 시군구는 서울시청 좌표 fallback (37.5665, 126.978)

## BR-9. HIRA sgguCd 매핑
- 행안부 sggCode(5자리)와 HIRA sgguCd(6자리)는 별도 코드 체계
- 실제 API 호출로 검증된 매핑 테이블 사용 (서울 25개 자치구)
- 매핑 미등록 시 인프라/안전 점수 "데이터 부족" 처리

## BR-10. 캐시 정책
- 실거래가: 24시간 TTL
- 시설 정보 (병원/약국): 7일 TTL
- 대기질: 1시간 TTL
- 행정구역 코드: 30일 TTL
- 외부 API 실패 시 만료된 캐시도 사용 (stale fallback, isStaleCache=true 표시)

## BR-11. 민감 지표 제외
- 범죄율 등 지역 낙인 유발 지표는 점수 산정에서 명시적 제외
- 안전 점수는 응급의료 접근성만 사용
- 학교 서열화, 학군 순위 미제공

## BR-12. 면책 안내
- 모든 리포트에 면책 문구 포함:
  "본 리포트는 공공 데이터에 기반한 참고 자료입니다. 부동산 투자 추천이나 절대적 판단을 제공하지 않습니다."
